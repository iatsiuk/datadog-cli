# Foundation: config, auth, root command, output formatting

## Overview

Establish the core infrastructure for datadog-cli: configuration management,
authentication via DD_API_KEY/DD_APP_KEY/DD_SITE environment variables,
root cobra command with global flags, and reusable output formatting (table/JSON).

This plan must be completed before any feature group plans.

Uses `github.com/DataDog/datadog-api-client-go/v2` as the official Datadog API client
and `github.com/spf13/cobra` for CLI framework.

## Context

- Entry point: `cmd/datadog-cli/main.go` (exists, imports `internal/cmd`)
- No `internal/` packages exist yet
- Go module: `github.com/iatsiuk/datadog-cli`, Go 1.25
- Build: `make build` (runs linter), `make test`

## Technical Details

### Config struct

```go
type Config struct {
    APIKey string
    AppKey string
    Site   string // default: "datadoghq.com"
}
```

### Client creation pattern

```go
func New(cfg Config) *datadog.APIClient {
    ddCfg := datadog.NewConfiguration()
    ddCfg.SetUnstableOperationEnabled("v2.SearchEvents", true) // etc.
    ctx := datadog.NewDefaultContext(context.Background())
    // set keys via context
    return datadog.NewAPIClient(ddCfg), ctx
}
```
