# APM: spans search, services, retention filters, span-based metrics

## Overview

Implement the `apm` command group for datadog-cli, providing CLI access to all
Datadog APM/Tracing APIs. Covers searching spans, listing services, managing
APM retention filters, and span-based metrics.

Uses `github.com/DataDog/datadog-api-client-go/v2` (datadogV2).

Depends on: foundation plan.

## Context

- SDK APIs used:
  - `datadogV2.SpansApi`: ListSpans, ListSpansGet, AggregateSpans (+ pagination variants)
  - `datadogV2.APMApi`: GetServiceList
  - `datadogV2.APMRetentionFiltersApi`: CRUD retention filters, reorder
  - `datadogV2.SpansMetricsApi`: CRUD span-based metrics

## Technical Details

### CLI structure

```
dd apm search --query "service:web-store" --from "now-1h" --to "now" --limit 50
dd apm tail --query "service:web-store env:prod"
dd apm aggregate --query "service:web" --compute "count" --group-by "@http.status_code"
dd apm services --env prod
dd apm retention-filter list|show|create|update|delete
dd apm span-metric list|show|create|update|delete
```

### Search output columns

Table mode: `TIMESTAMP | SERVICE | RESOURCE | DURATION | STATUS`

### Query syntax

Datadog span search syntax:
- `service:my-service`
- `resource_name:"/api/users"`
- `env:production`
- `@http.status_code:500`
- `duration:>1s`
- Boolean: AND, OR, - (negation)
