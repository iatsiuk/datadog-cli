# Security Monitoring: signals, rules, suppressions, filters

## Overview

Implement the `security` command group for datadog-cli, providing CLI access to
Datadog Security Monitoring APIs. Covers searching/triaging security signals,
managing detection rules, suppression rules, and security filters.

Uses `github.com/DataDog/datadog-api-client-go/v2` (datadogV2.SecurityMonitoringApi).

## Context

- SDK APIs used:
  - Signals: ListSecurityMonitoringSignals, SearchSecurityMonitoringSignals, GetSecurityMonitoringSignal, EditSecurityMonitoringSignalState, EditSecurityMonitoringSignalAssignee, EditSecurityMonitoringSignalIncidents
  - Rules: ListSecurityMonitoringRules, GetSecurityMonitoringRule, CreateSecurityMonitoringRule, UpdateSecurityMonitoringRule, DeleteSecurityMonitoringRule, ValidateSecurityMonitoringRule
  - Suppressions: ListSecurityMonitoringSuppressions, GetSecurityMonitoringSuppression, CreateSecurityMonitoringSuppression, UpdateSecurityMonitoringSuppression, DeleteSecurityMonitoringSuppression, ValidateSecurityMonitoringSuppression
  - Filters: ListSecurityFilters, GetSecurityFilter, CreateSecurityFilter, UpdateSecurityFilter, DeleteSecurityFilter
  - Findings: ListFindings, GetFinding, MuteFindings

- Files to create:
  - `internal/cmd/security.go` - main command + signals search/triage
  - `internal/cmd/security_test.go`
  - `internal/cmd/security_rule.go` - detection rules CRUD
  - `internal/cmd/security_rule_test.go`
  - `internal/cmd/security_suppression.go` - suppression rules CRUD
  - `internal/cmd/security_suppression_test.go`

## Technical Details

- All Security Monitoring APIs are in datadogV2.SecurityMonitoringApi (single API type)
- Signal severity values: info, low, medium, high, critical
- Signal state values: open, archived, under_review
- Rule types: log_detection, workload_security, cloud_configuration, infrastructure_configuration, application_security, signal_correlation
- Filter data types: logs, spans
- Suppression rules can have expiration dates (ISO 8601)
- Findings API may require unstable operation flag - check at implementation time
- `SearchSecurityMonitoringSignals()` uses POST with body, `ListSecurityMonitoringSignals()` uses GET with params
- Use `ListSecurityMonitoringSignals()` for simpler search pattern (consistent with logs/apm)
