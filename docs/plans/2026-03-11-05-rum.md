# RUM: events, applications, metrics, retention filters, replay, audiences

## Overview

Implement the `rum` command group for datadog-cli, providing CLI access to all
Datadog RUM (Real User Monitoring) APIs. Covers searching RUM events, managing
applications, RUM-based metrics, retention filters, session replay (playlists,
heatmaps, viewership), and audience management.

Uses `github.com/DataDog/datadog-api-client-go/v2` (datadogV2).

Depends on: foundation plan.

## Context

- SDK APIs used:
  - `datadogV2.RUMApi`: ListRUMEvents, SearchRUMEvents, AggregateRUMEvents, CRUD applications
  - `datadogV2.RumMetricsApi`: CRUD RUM-based metrics
  - `datadogV2.RumRetentionFiltersApi`: CRUD retention filters, reorder
  - `datadogV2.RumReplayPlaylistsApi`: CRUD playlists, manage sessions in playlists
  - `datadogV2.RumReplayHeatmapsApi`: CRUD heatmap snapshots
  - `datadogV2.RumReplaySessionsApi`: GetSegments
  - `datadogV2.RumReplayViewershipApi`: CRUD session watches, list watchers/history
  - `datadogV2.RumAudienceManagementApi`: connections, mappings, queries (accounts/users)

## Technical Details

### CLI structure

```
dd rum search --query "type:view" --from "now-1h" --limit 50
dd rum aggregate --query "type:error" --compute "count" --group-by "@view.url"
dd rum app list|show|create|update|delete
dd rum metric list|show|create|update|delete
dd rum retention-filter list|show|create|update|delete --app <id>
dd rum playlist list|show|create|update|delete
dd rum playlist sessions|add-session|remove-session
dd rum heatmap list|create|update|delete
dd rum session segments|watchers|watch|history
dd rum audience connections list|create|update|delete
dd rum audience mapping|query-users|query-accounts
```

### Search output columns

Table mode: `TIMESTAMP | TYPE | APPLICATION | VIEW | DURATION`
