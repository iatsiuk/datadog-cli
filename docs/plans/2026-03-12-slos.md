## Technical Details
- **SLO types**: `metric` (custom query) or `monitor` (based on existing monitors)
- **Thresholds**: `{timeframe, target, target_display, warning, warning_display}` -- pass as JSON string
- **Timeframes**: `7d`, `30d`, `90d`, `custom`
- **History**: returns SLI values, error budget remaining, uptime data per timeframe
- **Correction categories**: `Scheduled Maintenance`, `Other`, `Deployment`, `Infrastructure Issue`
- **Can-delete check**: returns list of dashboards/monitors referencing the SLO
