# Metrics: query, submit, metadata, tag configurations

## Overview

Implement the `metrics` command group for datadog-cli, providing CLI access to
all Datadog Metrics APIs. Covers querying timeseries/scalar data, submitting
metrics, managing metric metadata, tag configurations, and viewing metric assets.

Uses `github.com/DataDog/datadog-api-client-go/v2` (v2 MetricsApi) and
`github.com/DataDog/datadog-api-client-go/v2/api/datadogV1` (datadogV1 for query/search/list/metadata -- no V2 equivalents).

Depends on: foundation plan.

## Context

- SDK APIs used:
  - `datadogV1.MetricsApi`: QueryMetrics, ListMetrics, ListActiveMetrics, GetMetricMetadata, UpdateMetricMetadata, SubmitMetrics, SubmitDistributionPoints
  - `datadogV2.MetricsApi`: QueryTimeseriesData, QueryScalarData, SubmitMetrics, ListTagConfigurations, CreateTagConfiguration, UpdateTagConfiguration, DeleteTagConfiguration, ListTagsByMetricName, ListVolumesByMetricName, ListMetricAssets, EstimateMetricsOutputSeries, CreateBulkTagsMetricsConfiguration, DeleteBulkTagsMetricsConfiguration

## Technical Details

### CLI structure

```
dd metrics query --query "avg:system.cpu.user{*}" --from <unix> --to <unix>
dd metrics search --query "system.cpu"
dd metrics list --from <unix>
dd metrics scalar --query "avg:system.cpu.user{*}" --from <unix> --to <unix>
dd metrics timeseries --query "avg:system.cpu.user{*}" --from <unix> --to <unix>
dd metrics submit --metric "custom.metric" --type gauge --points "1234567890:42.0" --tags "env:prod"
dd metrics metadata show <name>
dd metrics metadata update <name> --type gauge --description "CPU usage" --unit percent
dd metrics tag-config list|show|create|update|delete
dd metrics tags <name>
dd metrics volumes <name>
dd metrics assets <name>
dd metrics estimate <name>
```

### Query output columns

Table mode: `TIMESTAMP | VALUE`
